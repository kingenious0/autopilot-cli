# Contributing

Thanks for contributing to Autopilot CLI!

## Quick Start

1. **Fork the repo** on GitHub.
2. **Clone your fork**:
   ```bash
   git clone https://github.com/YOUR_USERNAME/autopilot-cli.git
   cd autopilot-cli
   ```
3. **Create a branch**: 
   ```bash
   git checkout -b feature/your-change
   ```
4. **Make changes** and test.
5. **Run tests**:
   ```bash
   node --test
   ```
6. **Open a pull request**.

## Code Style

- Use **CommonJS** modules.
- Keep functions small and focused.
- Prefer clear logging and error handling.
- Avoid breaking existing CLI behavior.

## Commit Messages

Use conventional commits when possible:

- `feat: ...` for new features
- `fix: ...` for bug fixes
- `docs: ...` for documentation
- `chore: ...` for maintenance

## Documentation

If behavior changes, please update:
- `README.md`
- Relevant docs under `docs/` or `autopilot-docs/content/docs/`
